Goedel.Recrypt Namespace

Mesh/Recrypt Protocol
Classes
  ClassDescription
AddMemberRequest
Add a member to a recryption group (not currently used)
AddMemberResponse
Reports the success or failure of a AddMember request. The operation either succeeds or fails, there are no returned parameters
CreateGroupRequest
Request creation of a recryption group. The only request parameter describes the group to be created.
CreateGroupResponse
Reports the success or failure of a CreateGroup request. The operation either succeeds or fails, there are no returned parameters
Extension
Extension methods for recryption classes.
GetGroupRequest
Obtain recryption group data
GetGroupResponse
Obtain recryption group response.
GetKeyRequest
Request the current recryption key for the specified recryption group. NB: At present the group key is NOT authenticated and thus a MITM can perform a key substituition attack. This will be fixed in the next release.
GetKeyResponse
Response to get key request.
MemberEntry
Describes a member of a recryption group
MeshRecrypt
Mesh profile for Recrypt applications.
NoRecryptionRecord
There is no recryption record for the specified key combination
ProfileInvalid
Profile not valid
RecryptDataRequest
Request that the service provide a recryption result for the specified encrypted data and return it encrypted under the user's public key.
RecryptDataResponse
Result of recryption request.
RecryptDevicePrivate
Private data specific to the device
RecryptDevicePublic
Contains public device description. This is unused as there is no requirement to disclose the binding of keys to devices.
RecryptException
Error occurred in Recrypt library
RecryptionGroup
Describes a recryption group and set of members.
RecryptionKey
Describes a recryption key with optional additional description.
RecryptPortal
Abstract interface to a service that supports the MeshPortal API calls. Mostly for useful in test code where the ability to switch between a direct and indirect portal connection is desirable.
RecryptPortalRemote
Connection to network service using HTTP client.
RecryptProfile
Application profile for Confirm
RecryptProfilePrivate
Private portion of profile. This is unused. Decryption keys are provisioned on a per device basis to allow the use of co-generation techniques.
RecryptProtocol
Mesh/Recrypt Protocol Schema Mesh/Recrypt group administration protocol
RecryptRequest
Base class for all request messages.
RecryptResponse
Base class for all response messages. Contains only the status code and status description fields.
RecryptService
The new base class for the client and service side APIs.
RecryptServiceClient
Client class for RecryptService.
RecryptServiceProvider
Client class for RecryptService.
SessionRecryption
Manage a set of application sessions that are recryption sessions bound to a single personal session. This allows for methods such as 'get set of candidate keys'
UpdateGroupRequest
Request an update to a recryption group. Note that the update process is currently limited to 'strike and replace'. This is likely to become cumbersome if groups with very large numbers of entries are being maintained. It is likely that a future version of the protocol will support update requests that implement commonly occurring tasks such as updates to add a new encryption key, etc.
UpdateGroupResponse
Reports the success or failure of a UpdateGroup request. The operation either succeeds or fails, there are no returned parameters
UpdateMemberRequest
Update a recryption group member entry (not currently used)
UpdateMemberResponse
Reports the success or failure of a UpdateMember request. The operation either succeeds or fails, there are no returned parameters
UserDecryptionEntry
Decryption entry for a particular user and group key
